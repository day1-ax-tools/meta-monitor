메타 모니터 브리핑 보드

메인 세션의 질문과 답변 흐름, 산출물, Git 상태를 관찰해 현재 상황과 다음 행동을 메타 레벨에서 설명하는 보조 현황판입니다. 숨겨진 모델 추론 과정은 관찰하지 않습니다.

실시간 모니터 콘솔

메인 세션 이벤트를 우선 관찰하고, 필요할 때만 추가 질문을 큐로 보냅니다. 모니터 답변은 실시간 스트림으로 누적합니다.

연결 준비
시스템 모니터 콘솔을 준비했습니다. 브리지가 연결되면 최근 메인 세션 이벤트와 모니터 답변이 자동으로 표시됩니다. 쓰기 범위: meta-monitor/**

메인 세션 이벤트: meta-monitor/main-session-events.jsonl · 추가 질문 큐: meta-monitor/questions.jsonl · 입력 JSON: meta-monitor/monitor-input.latest.json · 답변 파일: meta-monitor/meta-advice.md · 이벤트 스트림: /api/events

최근 관찰 이벤트

아직 메인 세션 이벤트를 읽지 않았습니다.

출처: meta-monitor/main-session-events.jsonl

최근 모니터 답변

아직 모니터 답변을 읽지 않았습니다.

출처: meta-monitor/meta-advice.md

출력 확인 방식

메인 세션이 진행될 때마다 보이는 질문, 답변 요약, 파일 변경, 검증 결과를 meta-monitor/main-session-events.jsonl에 남깁니다. 모니터 CLI는 이 이벤트를 우선 읽어 meta-monitor/monitor-input.latest.json을 만들고, 그 결과를 meta-monitor/meta-advice.md에 씁니다. 추가 질문은 보조 경로로 meta-monitor/questions.jsonl에 쌓입니다.

현재 단계

CLI 온보딩 진입 페이지와 메타 모니터 계약을 다듬는 중입니다.

현재 목표

사용자가 CLI 작업으로 넘어가기 전에 선택형 모니터 경험을 이해할 수 있게 만듭니다.

다음 행동

이 브리핑 보드를 시각적으로 확인한 뒤, 현재 온보딩 변경사항의 커밋과 푸시 여부를 결정합니다.

작업 상태

수정 파일 11개와 모니터 산출물

온보딩 변경사항은 아직 커밋 전이며, 새 모니터 산출물은 meta-monitor/ 아래에 격리되어 있습니다.

예상 모니터 컨텍스트 사용량은 낮습니다. 아직 인계 문서는 필요하지 않습니다.

결정 사항

  • HTML 모니터링 화면의 이름은 브리핑 보드로 둡니다.
  • 메타 모니터는 사용자가 명시적으로 요청한 뒤에만 시작합니다.
  • 초기 구현 경로는 skill과 산출물 계약으로 시작합니다.

열린 질문

  • 이 페이지를 나중에 plugin 자산으로 배포할지 결정해야 합니다.
  • 학습자에게 추천할 기본 provider와 model 조합을 정해야 합니다.
  • 실시간 모니터 세션을 온보딩의 어느 시점에 소개할지 정해야 합니다.

위험 요소

  • 정적 HTML은 모든 브라우저 환경에서 파일을 직접 쓸 수 없습니다.
  • 모니터 세션은 meta-monitor/** 밖에 쓰지 않아야 합니다.
  • 인계 문서가 누적되면 컨텍스트 오염이 생길 수 있습니다.

업무 지도

역할, 목적, 성과, 업무 영역, 작업 분해를 한 화면에서 확인하는 탭입니다. 상세 시각화가 필요하면 meta-monitor/visualizations/work-map-dashboard.html로 분리합니다.

지도 구성

층위내용
목적사용자가 어떤 결과를 만들기 위해 일하는지 정리합니다.
역할사용자의 책임, 협업 대상, 판단 권한을 기록합니다.
업무 영역반복되는 업무 묶음을 나누고 우선순위를 봅니다.
작업 단위자동화 후보로 내려갈 수 있을 만큼 작게 쪼갭니다.

현재 신호

  • 업무 인터뷰는 아직 실제 사용자 업무 단계로 진입 전입니다.
  • 앞단 환경설정과 모니터 경험을 먼저 안정화하고 있습니다.
  • 다음 인터뷰에서는 역할 설명에서 시작해 자동화 후보까지 위에서 아래로 내려갑니다.

미션 백로그

자동화 후보, 구현 후보, 검증이 필요한 후보를 모아두는 탭입니다. 상세 시각화가 필요하면 meta-monitor/visualizations/mission-dashboard.html로 분리합니다.

분류 기준

바로 실행

입력과 출력이 명확하고, 위험이 낮고, 검증 방법이 분명한 작업입니다.

설계 필요

업무 가치는 있지만 데이터, 권한, 프로세스 경계가 더 필요한 작업입니다.

보류

현재 권한, 비용, 리스크, 검증 부담 때문에 나중에 다루는 작업입니다.

의사결정 지도

선택지, 장단점, 위험, 되돌릴 수 있는지 여부를 정리하는 탭입니다. 상세 시각화가 필요하면 meta-monitor/visualizations/decision-map.html로 분리합니다.

현재 결정

  • 브리핑 보드는 CLI를 대체하지 않고, CLI 작업을 관찰하고 보조합니다.
  • 설정은 우상단 대화상자로 옮기고, 기본 화면은 작업 흐름에 집중합니다.
  • 화면 구조는 탭으로 나누어 모니터 대화와 현황 확인을 분리합니다.

남은 결정

  • 메타 모니터를 스킬, 플러그인, MCP 중 어디까지 제품화할지 정해야 합니다.
  • 초보자에게 실시간 모니터를 언제 보여줄지 정해야 합니다.
  • 세션 데이터 기반 학습 기능의 첫 범위를 정해야 합니다.

검증 관점

  • 질문 제출이 끊기지 않는지 확인합니다.
  • 모니터 쓰기 범위가 meta-monitor/**로 유지되는지 확인합니다.
  • 인계 문서는 항상 최신 파일 하나만 남는지 확인합니다.
